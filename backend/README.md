# 🧩 Backend (Serverless) — FSD-FULL

Полноценный backend, переписанный под **Vercel Serverless Functions**,  
основанный на **Node.js + Express + Mongoose**, построенный по архитектуре  
**MVC + Services**, с авторизацией, задачами, ролями, приглашениями и автоинициализацией данных.

Backend полностью совместим с фронтендом проекта.

---

## 📁 Структура проекта

```bash
backend/
│
├── api/                 # Serverless-функции (каждый файл = эндпоинт Vercel)
│   ├── index.js         # Главная точка входа (Express → Serverless)
│   ├── auth/            # /api/auth
│   ├── profile/         # /api/profile
│   ├── priority/        # /api/priority
│   ├── status/          # /api/status
│   └── todos/           # /api/todos
│
├── controllers/         # Контроллеры для обработки запросов
├── services/            # Сервисы (бизнес-логика + работа с MongoDB)
├── models/              # Mongoose-схемы (User, Todo, Status, Priority…)
├── middleware/          # JWT-проверка, защита приватных маршрутов
├── utils/               # Хелперы: email, токены, генераторы
│
├── lib/
│   ├── server.js        # Создание Express-приложения (без app.listen)
│   └── auth.js          # JWT middleware в serverless-формате
│
├── initDefaults.js      # Автоинициализация статусов/приоритетов
├── package.json         # Зависимости + настройка serverless окружения
└── vercel.json          # Конфигурация деплоя на Vercel
```

---

## 🚀 Особенности serverless backend

- Работает **без app.listen()**
- Каждый эндпоинт вызывается Vercel как отдельная функция
- Express полностью сохранён (через `serverless-http`)
- **MongoDB подключается как singleton** — повторных подключений нет  
- Абсолютно все API сохранены (фронтенд работает БЕЗ изменений)
- Лёгкая масштабируемость и мгновенный cold-start

---

## 🔐 Авторизация

- Регистрация  
- Логин  
- **JWT-токены** для доступа  
- Защищённые маршруты через middleware  
- **Google OAuth 2.0** — получение email / имени / аватара  
- Поддержка invite-логики при Google-входе  
- Автоматическое создание нового пользователя при первом OAuth-входе  

---

## 📝 Работа с задачами (Todos)

**Полный функционал:**

- Создание задач  
- Редактирование  
- Удаление  
- Привязка задачи к:
  - статусу
  - приоритету
  - владельцу (ownerId)
- Поля `vital`, `image`, `completedAt`, `createdAt`
- Обновление `completedAt` автоматически при смене статуса на *Completed*
- Если роль пользователя `view` → любые изменения запрещены

---

## 🟦 Статусы и приоритеты

- Модели `Status` и `Priority`
- Стандартные CRUD-операции
- Автоматическое создание дефолтных значений через `initDefaults.js`
- Защита от удаления дефолтных статусов и приоритетов

---

## ✉️ Система приглашений (Invites)

- Генерация приглашений
- Назначение ролей: `edit` / `view`
- Отправка email через SMTP
- Переход по ссылке: `/register?invite=TOKEN`
- Привязка приглашённого к workspace (ownerId)
- Добавление нового `Member` после регистрации
- Автоудаление использованного invite

---

## 👥 Роли пользователей

Доступные роли:

- `owner` — владелец workspace  
- `edit` — может управлять задачами  
- `view` — только просмотр  

Ограничения:

- `view` не может создавать/редактировать/удалять задачи  
- `owner` может приглашать участников

## 📦 Установка и локальный запуск

### 1️⃣ Установить зависимости

```bash
cd backend
npm install
```

### 2️⃣ Локальный запуск в режиме Vercel

```bash
npm run vercel-dev
```

API будет доступно по адресу:

```
http://localhost:3000/api/
```
